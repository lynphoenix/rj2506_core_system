# RJ2506 全局架构与任务拆解总表

**文档状态**: V3.0 (新增顶层大脑节点与完整 8 段式管线)
**目标**: 提供整个双臂轮式机器人的全局需求、核心难点以及模块化的任务清单，作为整个项目文档树的**入口导航页**。

---

## 1. 整体任务描述与核心需求

**整体任务描述 (The Overall Mission)**：
RJ2506 是一款双臂轮式复合机器人（14轴：双臂各6轴 + 2驱动轮），旨在解决工厂半结构化环境下的高精度柔性装配、堆垛与搬运问题。目前主要聚焦并适配两大核心工业应用场景：

### 场景 1：冲压车间大框码料 (大料片搬运与高精度码垛)
*   **任务目标**: 从下料平台双手同时抓取 5-6 片大料片，搬运并精准码放到 1.2m×1m×0.75m 的大框中（满载2吨，垂直轮换）。节拍要求 15-18s/次。
*   **业务动作流程 (Physical Workflow)**:
    1. 机器人在待机区接收到“下料平台待抓取”的生产信号。
    2. 机器人移动到冲压机台的下料平台前。
    3. 双手伸出，抵住平台边缘把自己的位置卡死对齐。
    4. 准确抓住 5-6 片堆叠好的大料片。
    5. 机器人转身移动到专门存放料片的大框旁边。
    6. 将手中的大料片放入大框内。由于大框边缘限制，放置必须非常平稳精准，不能磕碰大框。
    7. 确认放到底部后松手，双臂退回安全姿态，完成一次循环。

### 场景 2：冲压车间小框换框 (料框替换与堆叠搬运)
*   **任务目标**: 推动空框替换冲压机台满框，并将装满小零件的满框双手搬运至中转站进行 5层×2排 的高精度码垛。节拍要求 5分钟/次。
*   **业务动作流程 (Physical Workflow)**:
    1. 接收到冲压机台“料框已满”的信号。
    2. 机器人先前往空框区，拿取一个空框作为替换用。
    3. 带着空框移动到冲压机台下料口。
    4. 机器人用手里的空框顺着机台导轨往里推，利用空框把里面装满零件的满框给直接顶出来（完成物理替换）。
    5. 双臂从两侧牢牢抱起被顶出来的满框。
    6. 搬运满框前往中转站。
    7. 在中转站进行 5层×2排 的高精度码垛，要求叠放平稳，重心不能偏移导致倒塌。
    8. 放稳后松手退回，准备下一次换框。

**硬件载体总结**: RJ2506 差速底盘 + 躯干控制器 + 双机械臂 (单臂负载 15kg，协同 30kg) + 左右平行夹爪 + 头部全局相机(OAK-D) + 手腕局部相机。
*   **计算平台**: NVIDIA Jetson Orin (由于现有任务冲突，Phase 0/1 阶段暂不刷入 PREEMPT_RT 实时内核)。
*   **业务场景**: 工厂半结构化环境下的纸箱堆垛、轮胎抱持、料片抓取。
*   **底层通信**: CANopen (1000Hz, 控制电机) + RS-485 (50Hz, 控制夹爪)。

## 2. 全局软件架构 (八段式控制管线)

由于端到端模型 (ACT/DP) 无法保证导航和物理接触的绝对安全，且底盘停靠必然存在 $\pm 5cm$ 的宽容误差，系统采用由**顶层大脑节点 (Brain Node)** 统一调度的**八段式控制管线**。

### 2.1 标准八段式管线定义
0.  **顶层大脑 (Brain Node / State Machine)**: 贯穿始终，负责串联以下所有阶段的流转、权限交接与异常恢复。
1.  **任务触发与备料 (Task Trigger & Prep)**: 接收 MES/PLC 信号，唤醒机器人，执行必要的取空框备料或从待机点启动。
2.  **宏观导航 (Nav2)**: 负责厂区大范围移动，停靠在工站前 $\pm 5cm$ 的宽容区域内。
3.  **物理顺应补偿 (WBC & Impedance Control)**: 放弃让底盘精调，利用双臂推拉料框等物理接触动作，配合力控强行“抹平”底盘误差。
4.  **大尺度抓取 (ACT / Diffusion Policy)**: 输入图像与本体感知，模型推理输出双臂协作轨迹，双手抓取/抱持目标。
5.  **高空特征记忆 (Feature Memory)**: 移动至目标位置并在放置点上方 15cm 处悬停，通过双目提取 3D 特征点，建立绝对锚点。
6.  **微插闭环 (Virtual IBVS)**: 视线彻底遮挡后接管控制权。利用高空记忆的特征点构建“虚拟图像雅可比”，抗遮挡盲插闭环并实时修正 6D 姿态。
7.  **触底检测与释放 (Touch-down & Release)**: 结合阻抗控制与电机电流（$\Delta I / \Delta t$）突变检测触底，完成后控制双侧夹爪同步安全释放。
8.  **复位与回程 (Reset & Return)**: 双臂收回行车姿态，底盘返回待机点或进行下一轮循环，向上位机上报任务完成。

### 2.2 场景 1 映射：大框码料 (大料片搬运)
这 8 段标准管线是如何完美匹配场景 1 的物理动作的：
*   **[0] 顶层大脑**: 接收冲压机台“下料平台待抓取”信号，启动大框码料状态机。
*   **[1] 任务触发与备料**: (对应动作 1) 从待机休眠模式唤醒，确认双臂处于安全行车姿态。
*   **[2] 宏观导航**: (对应动作 2) 底盘导航并宽容停靠在冲压下料平台前。
*   **[3] 物理顺应补偿**: (对应动作 3) 双手伸出抵住下料平台边缘，利用阻抗控制将自身位姿卡死对齐。
*   **[4] 大尺度抓取**: (对应动作 4) DP 模型视觉输出，双手准确抓取 5-6 片大料片。
*   **[5] 高空特征记忆**: (对应动作 5) 转身移动到大框旁边，在大框放置点上方 15cm 处悬停，记录大框边角特征。
*   **[6] 微插闭环**: (对应动作 6) 带着宽大的料片往下放，视线全盲，切入虚拟 IBVS 实时修正 6D 姿态，确保料片平稳入框不磕碰。
*   **[7] 触底检测与释放**: (对应动作 7 前半段) 监听 Z 轴电流突变感知放到底部，双侧夹爪同步松手。
*   **[8] 复位与回程**: (对应动作 7 后半段) 双臂退回安全姿态，完成一次循环，系统复位。

### 2.3 场景 2 映射：小框换框 (料框替换与码垛)
这 8 段标准管线是如何解决场景 2 的换框与 5 层高精度码垛的：
*   **[0] 顶层大脑**: 接收“料框已满”信号，启动换框与码垛状态机。
*   **[1] 任务触发与备料**: (对应动作 1-2) 唤醒后，大脑指引底盘先前往空框区，抓取一个空框作为替换“弹药”。
*   **[2] 宏观导航**: (对应动作 3) 带着空框移动到冲压机台下料口停靠。
*   **[3] 物理顺应补偿**: (对应动作 4) 用手里的空框顺着机台导轨往里推，力位混合控制确保不卡死，同时顺势把满框顶出来（物理替换）。
*   **[4] 大尺度抓取**: (对应动作 5) DP 模型感知顶出的满框位姿，双手从两侧牢牢抱起满框。
*   **[5] 高空特征记忆**: (对应动作 6 及 7 前半) 搬运满框前往中转站，在当前堆叠层上方悬停，提取下方料框的榫卯特征。
*   **[6] 微插闭环**: (对应动作 7 后半) 往下放置时满框挡住视线，依靠虚拟雅可比矩阵修正 Pitch/Roll 倾斜，防止 5 层码垛重心偏移倒塌。
*   **[7] 触底检测与释放**: (对应动作 8 前半) 接触力矩达到峰值判定码放稳固，松开双臂。
*   **[8] 复位与回程**: (对应动作 8 后半) 退回行车姿态，准备下一次换框任务。

这是不需要懂算法的实施人员必须做完的前置体力活。

*   **[0.1] 环境与连线**: 系统刷机（不含实时补丁）、ROS 2 安装、外设连通性测试。（注：因 OAK-D 标准版无触发引脚，放弃硬件连线同步）。
*   **[0.2] 相机外参标定**: 跑脚本算出 `camera_to_flange` 外参矩阵 ($T_{extrinsic}$)。
*   **[0.3] 绝对精度验证**: 制作物理探针(TCP)，手工戳出 4 个验证点的世界绝对坐标，与视觉计算对比，确保 RMSE < 1.5像素。
*   **[0.4] 盲插定妆照采集**: 人工将零件完美插入卡槽后抬高 15cm，拍摄作为后续 IBVS 控制的 2D 目标金标准 ($s_{target}^*$)。

### Phase 1: 核心硬骨头算法模块攻坚 (研发重点)
此阶段负责攻克卡脖子的技术难题，各模块可独立开发。

*   **[模块 1] 双臂全身协调控制 (Whole-Body Control, WBC)**: (运动学核心)
    *   *核心挑战*: RJ2506 是一个 14 轴的复杂系统（双臂各 6 轴 + 底盘 2 轴）。如何在狭窄的工厂车间内，解算双臂的干涉避障，并计算双手抓取大料片或抱持满框时的闭链运动学（Closed-chain Kinematics）。
*   **[模块 2] 顶层状态机与任务编排 (Brain Node)**: (架构中枢)
    *   *核心挑战*: 设计一个健壮的 ROS 2 Lifecycle Node 或 Behavior Tree，管理各个算法节点的生命周期，处理“抓取失败”、“找不到特征”、“触底碰撞异常”等各种边缘情况，实现优雅的异常降级与恢复。
*   **[模块 5] C++ 守护进程与软件时间同步**: (底层基石，优先级最高)
    *   👉 **[开发设计文档：03-Module5-Cpp-Daemon-Time-Sync.md](./03-Module5-Cpp-Daemon-Time-Sync.md)** (已完成)
    *   *核心挑战*: 放弃硬件同步后，如何在 1000Hz 纯净线程里管理 CANopen，并利用**环形缓冲区 (Ring Buffer) + 样条插值**解决图像和电机的“因果混淆”问题。
*   **[模块 3] 双目特征记忆提取器**: (视觉核心)
    *   👉 **[开发设计文档：05-Module3-Feature-Memory.md](./05-Module3-Feature-Memory.md)** (已完成)
    *   *核心挑战*: 设计双轨特征提取（传统几何算法保障极低延迟，深度学习 SuperPoint/LightGlue 利用空闲 GPU 作为鲁棒性托底），在恶劣光照下稳定提取 $N \ge 4$ 个 3D 特征点。
*   **[模块 4] 虚拟 IBVS 视觉伺服**: (控制核心)
    *   👉 **[开发设计文档：04-Module4-Virtual-IBVS-Math.md](./04-Module4-Virtual-IBVS-Math.md)** (已完成)
    *   *核心挑战*: 包含深度的 6x6 图像雅可比矩阵推导，防 Pitch/Roll 倾斜发散，伴随矩阵速度映射。
*   **[模块 6] 力控、触底检测与物理顺应 (Compliance)**: (物理交互核心)
    *   👉 **[开发设计文档：06-Module6-Impedance-Control.md](./06-Module6-Impedance-Control.md)** (待编写)
    *   *核心挑战*: 怎么在只支持位置/速度模式的 CANopen 驱动器外面，套一层阻抗控制模型（模拟弹簧）；如何依靠电机电流突变 $\Delta I / \Delta t$ 识别插槽触底，并安全触发夹爪释放机制。
*   **[模块 7] 双夹爪同步通信与错误恢复**: (末端控制)
    *   *核心挑战*: 两个夹爪通过 RS-485 级联，工作在 50Hz。在双臂抱持轮胎或端大框时，如果一个夹爪由于通信丢包导致释放延迟，将直接导致物体掉落或机械臂拉扯。如何设计一个强同步机制。

### Phase 2: AI 训练与全链路联调
*(待 Phase 1 的底层基建 100% 跑通并验证后开启)*
👉 **[开发设计文档：07-Phase2-AI-Training-And-Integration.md](./07-Phase2-AI-Training-And-Integration.md)** (已完成)

*   **[2.1] 遥操作数据采集与处理 (Data Collection)**: 采用 Leader-Follower 模式录制 50 条真机专家轨迹。利用 C++ Daemon 环形缓冲区保证动作与图像时间戳严格对齐。之后通过 IsaacLab 域随机化扩充 10000 条数据。
*   **[2.2] ACT 模型训练与部署 (Training & Deployment)**: 训练 ACT 模型直接输出 14 轴关节空间位姿 ($q$)，以避免逆运动学奇异点。通过 TensorRT 在 Orin 端实现 $< 30ms$ 低延迟推理。
*   **[2.3] 大脑节点 (Brain Node) 串联**: 构建强健状态机（Behavior Tree），将 Nav2 -> WBC -> ACT -> 特征记忆 -> IBVS 盲插 -> 触底释放 完整管线打通，并负责异常重试与安全降级。

### Phase 3: 工程化落地与自动标定 (Production Readiness)
*(系统在真实工厂 7x24 小时运行的生命周期保障)*

这部分解决“实验室 Demo”到“工业产品”的长尾问题，重点回答一个核心质疑：**既然能做自动标定，为什么 Phase 0 还要安排实施人员做极其繁琐的手动标定？**

*   **[3.1] 标定逻辑的“第一性原理” (Manual vs. Auto Calibration)**:
    *   **Phase 0 (手动极限标定) 是地基**：自动标定本质是非线性优化 (LM 算法) 的漂移微调，**极其依赖准确的初始值 (Initial Guess)**。如果初始值是瞎蒙的，自动标定极易陷入反物理直觉的“局部极小值”。同时，只有人工拖拽出 30+ 个大倾角极限姿态（充分激励），才能解耦出高精度的 6D 旋转外参。此外，必须用物理探针 (Ground Truth) 引入独立于系统之外的绝对第三方刚性基准，打破“电机读数与视觉互相证明自己”的数学死循环。
    *   **Phase 3 (自动免工具复校准) 是维护**：在 Phase 0 打好绝对地基的前提下，机器人在工厂每天唤醒后，仅需花 10 秒钟通过双臂互相触碰或观测基座固定二维码，即可计算出微小的重投影误差，全自动补偿因车间温差形变、轻微碰撞导致的那几毫米外参漂移。
    *   **结论**：没有手动标定，自动标定就是瞎子摸象、空中楼阁；没有自动标定，手动标定的精度在恶劣车间里撑不过三天。
*   **[3.2] 自动化运维与多进程守护 (DevOps)**: 将 ROS 2 与 TensorRT 环境全面 Docker 化，实现 OTA 远程一键热更新。引入进程守护机制，在 CANopen 或视觉节点崩溃时，立刻拉起并在拉起期间控制双臂紧急刹车至安全位。
*   **[3.3] 数据飞轮与影子模式 (Data Engine)**: 当阻抗控制检测到异常碰撞（电流突变 $\Delta I / \Delta t$ 激增）或插孔失败时，自动截取前后 10 秒的 HDF5 数据并回传至云端。实施人员后台接管纠正一次，算法组微调后下发，形成“失败 -> 收集 -> 微调 -> OTA”的长尾场景闭环。
