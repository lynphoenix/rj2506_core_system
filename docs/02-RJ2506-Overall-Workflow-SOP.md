## 任务一：安装基础系统与 ROS 2 环境 (暂不刷实时系统)

**目标**：为 Jetson Orin 安装必要的驱动和 ROS 2 运行环境。因为目前 Orin 还有其他计算任务冲突，我们**暂时不刷入 PREEMPT_RT 实时系统**。

**操作步骤**：
1. **安装 JetPack 6.0**：
   * 找一根 Type-C 数据线，一头插在你的笔记本（必须是 Ubuntu 系统）上，一头插在机器人背后的 NVIDIA Jetson Orin 的刷机口 (Recovery Port)。
   * 用针按住 Orin 主板上的 `REC`（恢复）小按钮不放，同时按一下 `RST`（重启）按钮，松开 `REC` 按钮进入刷机模式。
   * 在你的笔记本终端里，输入 `sdkmanager` 打开 NVIDIA SDK Manager 软件。
   * 选择 **JetPack 6.0** 并点击 Flash 刷入标准系统。中间会弹出一个框让你设置机器人的用户名和密码（比如设置成用户名 `rj2506` 密码 `123456`）。
   * **注意：千万不要勾选 PREEMPT_RT 补丁！**
2. **安装 ROS 2 基础环境**：
   * 刷机完成后，用一根网线把机器人连上公司的网。
   * 在机器人终端里，把这段脚本复制进去，按回车执行（这会全自动帮你装好 ROS 2 软件和相机驱动）：
     ```bash
     sudo apt update && sudo apt install curl -y
     curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
     echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
     sudo apt update && sudo apt install ros-humble-ros-base ros-humble-rmw-cyclonedds-cpp ros-humble-depthai-ros -y
     ```
3. **验收标准**：
   * 打开终端，输入：`ros2 --version`。如果能打印出版本号，说明基础环境搞定了。

## 任务二：手眼相机标定数据采集 (Eye-in-Hand)

**目标**：让算法算出“手腕相机”和“机械臂手抓（法兰）”之间的精确安装距离和角度。

**准备工作**：
* 找一块印有黑白方块的标定板（ChArUco Board，类似于二维码大集合）。
* 用双面胶或夹具，把这块板子**死死固定**在机器人面前的工作台上。绝对不能有任何晃动。

**操作步骤**：
1. **下载并编译标定软件**：
   * 打开机器人电脑，拉取视觉代码并使用 `colcon` 进行编译：
     ```bash
     mkdir -p ~/ros2_ws/src
     cd ~/ros2_ws/src
     git clone git@github.com:rj2506/rj2506_vision.git
     cd ~/ros2_ws
     colcon build --symlink-install
     source install/setup.bash
     ```
2. **打开标定软件**：在上面那个终端里输入：`ros2 run rj2506_vision eye_in_hand_collector`。屏幕上会弹出一个相机实时画面。
3. **解锁机械臂**：按下机器人控制手柄（示教器）背后的使能按键，开启“拖拽模式 / 柔顺模式”。
4. **开始扭秧歌（采集数据）**：
   * 你需要用手抓着机械臂，在标定板上方 20 厘米 ~ 40 厘米的空中，摆出各种**极其扭曲**的姿态（左歪、右斜、前倾、后仰）。
   * 保证每次停下来时，相机的画面里都能清晰看到那块黑白方块板。
   * 机械臂停稳后（手不要抖），按一下键盘上的 `C` 键。屏幕会提示“捕获成功：1/30”。
   * 继续换一个完全不同的姿态，再按 `C` 键。
   * **要求**：至少采集 **30 个**截然不同的姿态。倾斜角度越大越好，不要总是让相机垂直往下看！
5. **生成文件**：采集够 30 个点后，按 `S` 键。软件会卡住几秒钟，然后提示：`外参文件已保存至 /etc/rj2506/extrinsics/rj2506_extrinsics.yaml`。采集任务完成。

## 任务三：制作并标定“物理测量笔” (TCP 探针)

**目标**：我们要把机械臂变成一把极其精准的“三坐标测量尺”。为了做到这点，我们要给机械臂装一个“笔尖”，并算出这个“笔尖”相对于法兰的精确位置（TCP Offset）。

**准备工作**：
* 去工厂机床那边，找师傅车一个**金属圆锥体**（像一个大号铅笔头），顶端要非常尖锐。
* 用螺丝把这个金属圆锥体，死死锁在机械臂的末端法兰上。**用手掰一下，绝对不能有一丝晃动！**

**操作步骤 (自研四点标定法)**：
1. **找一个固定的靶点**：在桌子上用胶水固定一根朝上的大头针（或者找一个有明显尖锐凸起的固定零件）。
2. **启动终端标定程序**：在机器人电脑里，新开一个终端并运行自研工具：
   ```bash
   ros2 run rj2506_kinematics tcp_calibrator
   ```
3. **第一碰**：
   * 按下示教器背后的使能按键，开启拖拽模式，你用手拉着机械臂。
   * 用你装好的那个“金属笔尖”，去**极其精确地戳中**桌子上那根大头针的尖端。一定要针尖对针尖！
   * 对准后，在运行刚才命令的终端键盘上按下 `C` 键。屏幕会提示“成功捕获第 1 个姿态”。
4. **第二、三、四碰**：
   * 保持“针尖对针尖”不动，**把机械臂的姿态大幅度扭转一个角度**（比如把手肘抬高、或者往左歪）。
   * 确保探针尖端依然死死抵在大头针尖端上。再次按下键盘 `C` 键捕获第 2 个点。
   * 再换两个极其夸张的扭曲姿态，分别按下 `C` 键捕获第 3 和第 4 个点。
5. **验收标准**：
   * 按下键盘上的 `S` 键开始计算。
   * 终端屏幕会打印出解算出的 TCP 偏移量（XYZ），并给出一个 `RMSE`（均方根误差）。
   * **及格线**：RMSE 误差必须 `< 0.5 mm`。如果大于这个数（终端会报红色警告），说明你刚才手抖了没有戳准或者姿态变化太小，必须按 `Q` 退出然后从第 2 步重新戳！
   * 如果误差达标，文件会自动写入 `/etc/rj2506/kinematics/tcp_offset.yaml`。

## 任务四：采集验证相机的“绝对物理坐标” (Ground Truth)

**目标**：刚才任务二算出了相机安装位置，现在我们要用刚做好的“测量笔”，戳出物理世界的绝对坐标，交给算法工程师去验证相机算得准不准。

**准备工作**：
* 拿一块**全新**的板子（上面画着 4 个小圆点）。
* **严重警告：绝对不能用任务二里用过的那块黑白方块板！必须是一块全新的板子！**
* 把这块带 4 个圆点的验证板，平放在工作台上，用胶带贴死。

**操作步骤**：
1. **戳第 1 个点**：
   * 开启机械臂拖拽模式。
   * 用你的“金属笔尖”，去极其精确地戳中验证板上**左上角**那个圆点的圆心。
   * 屏住呼吸，手不要抖，看着电脑屏幕，在终端里输入命令：`ros2 topic echo /tf` （或者让旁边的程序员帮你用脚本记录一下当前法兰的坐标）。
   * 把这一长串数字，或者脚本打印出的 `Point 1: X=..., Y=..., Z=...` 记录在 Excel 表格的“点 1”里。
2. **戳其余 3 个点**：
   * 按照顺时针方向，用笔尖依次戳中右上角、右下角、左下角的圆心。
   * 在戳中每个点时，记录下机器人的坐标，分别填入 Excel 的“点 2”、“点 3”、“点 4”里。
3. **验收标准**：
   * 把拆下金属探针，把手腕相机装回去。
   * 把这份填好 4 个点坐标的 Excel 表格交给算法工程师。你的物理采集任务就结束了，剩下的他们会用脚本去验证误差是否 `< 1.5 像素`。

## 任务五：采集“盲插金标准”特征图

**目标**：机器人在最后 10 厘米瞎子摸象插零件时，需要一张“完美对准”时的标准照片作为参照物。你要帮机器人拍下这张“定妆照”。

**准备工作**：
* 找算法工程师要一下 `capture_golden_template.py` 脚本（通常在 `~/ros2_ws/src/rj2506_vision/scripts/` 目录下）。

**操作步骤**：
1. **物理对准**：
   * 在机器人的夹爪里塞入一个真实的零件。
   * 你用手拉着机械臂（拖拽模式），像人眼穿针引线一样，**极其完美地把零件插入工作台上的装配卡槽里**。这必须是严丝合缝、最完美的状态。
2. **垂直抬升**：
   * 在示教器上切换到【直线/笛卡尔运动模式】。
   * 控制机械臂，只沿着 Z 轴（正上方），垂直向上抬起整整 **15 厘米**。停住！不要动其他任何方向。
3. **拍定妆照**：
   * 在电脑终端上进入刚才找到的目录，运行命令：`python3 capture_golden_template.py`。
   * 这时相机的补光灯会闪烁一下，拍下一张照片，存到了你的主目录下。
4. **验收标准**：
   * 打开刚才生成的 `golden_template.jpg` 照片。
   * 照片的画面必须非常清晰，没有任何运动模糊。
   * 如果这张照片看上去是歪的，说明你刚才垂直抬起的时候手抖了，或者笛卡尔坐标系的 Z 轴没对准。必须重拍！

## 任务六：(算法组) 验证一：软件时间插值 (Software Time-Sync) 误差验证

**目标**：这是一份软件测试“验收单”。证明在 C++ 守护进程的 `RingBuffer` 插值补偿下，图像时间戳与电机姿态的最大因果错位被压榨到了 $< 10ms$ 以内。

### 6.1 验证步骤与命令
1.  **物理测试环境**：
    *   在机器人面前放置一个运行着高刷新率（> 120Hz）数字秒表的显示器。
    *   **非常重要**：操作员手握机械臂，以**最高速度**随机挥舞（无负载），模拟端到端训练时的极限角速度。
2.  **启动 C++ 守护进程 (Daemon)**：
    *   确保 Daemon 开启了 1000 帧环形缓冲区，并正在监听相机的 ROS 2 图像话题以执行时间戳倒推。
3.  **运行 Python 分析脚本**：
    *   在高速挥舞中，从话题录制连续 1000 帧（对应 30Hz 相机约 30 秒）的打包数据（包含图像和插值后的电机角度）。
    ```bash
    python3 scripts/verify_software_sync.py --samples 1000 --output sync_report.png
    ```

### 6.2 `verify_software_sync.py` 核心验证逻辑
1.  **因果一致性 (Causal Consistency) 检查**：
    *   脚本抽取 10 张画面极其模糊的高速运动照片。
    *   通过 OCR 识别显示器上的秒表读数 $T_{phys}$。
    *   对比这段数据在 HDF5 中绑定的推算出的电机编码器时间 $T_{motor\_interpolated}$。
    *   **合格线 (Pass)**：两者在时间轴上的相对滑动（Drift）必须保持在一个恒定的常数（即 USB 的固定延迟 $\Delta T$）。其上下波动的方差（抖动误差）**必须 $< 10ms$**。如果看到大于 30ms 的抖动，说明插值算法写错了，立刻打回去重做 03-Module5，否则 AI 必废。

## 任务七：(算法组) 验证二：6D 运动映射矩阵重投影误差验证

**目标**：在没有昂贵的激光跟踪仪 (Laser Tracker) 的简陋条件下，证明任务二算出的 `camera_to_flange` (YAML) 外参，和任务三算出的 TCP 物理坐标是**“自洽且精准的”**。我们要证明这些数字能完美将物理世界的 $XYZ$ 咬合在 2D 像素上。

### 7.1 验证原理：为什么叫做“重投影误差 (Re-projection Error)”？

这是一个非常直观的物理与几何的双重校验。它将“盲人摸象”转化为可见的像素差异。

**基本思想**：
假设我们已经知道了桌子上 4 个圆点在真实物理世界里的绝对坐标（$P_{world}$，实习生用尖锐探针一毫米一毫米戳出来的）。
如果相机的安装位置（外参 $T_{flange2cam}$）是完美精准的，机械臂的编码器读数也是完美精准的。那么当机械臂移动到任何一个新姿态时：
1. **数学脑补**：我们完全可以只靠坐标系矩阵乘法，加上相机的内参透视原理，在代码里“算”出这 4 个圆点**理论上应该出现在照片的哪几个像素上**（例如算出在像素 `(u=320.5, v=240.2)`）。这叫 $s_{virtual}$。
2. **现实打脸**：我们立刻拍下一张真实的物理照片。用图像处理算法在这张照片上寻找那 4 个圆点**实际上出现在哪个像素位置**（例如实际在 `(u=323.0, v=238.0)`）。这叫 $s_{real}$。
3. **裁决**：我们计算 $s_{virtual}$（脑补像素）和 $s_{real}$（真实像素）之间的欧氏距离。

**误差来源分析**：
这不仅是在验证外参，这是一个**全链路照妖镜**：
*   如果你的 TCP 探针戳歪了（Ground Truth 根本不对），脑补的像素一定会偏。
*   如果你的外参 $T_{flange2cam}$ 算错了，相机光心的绝对位置就错了，脑补的像素一定会偏。
*   如果机械臂的本体很拉跨（比如因为自重下垂形变导致正运动学读数骗了你），脑补的像素也会偏。

只要这个误差极小，就证明整套物理基建（本体刚性 + 探针精度 + 外参标定）完美合格！

### 7.2 验证步骤与命令

1.  **准备 Ground Truth 数据**：
    *   拿来实习生在执行“任务四”交上来的那份 Excel 表格。里面包含了他们用 TCP 物理探针亲手戳出来的 4 个验证点在基座坐标系 (`base_link`) 下的绝对 3D 坐标 $P_{world}^j$。
    *   **再次警告**：如果这 4 个点是用之前标定的黑白方块板测出来的，直接宣布无效，让实习生换一块板子重戳！
2.  **准备外参文件**：
    *   确认 `/etc/rj2506/extrinsics/rj2506_extrinsics.yaml` 文件存在且由最新的 30 个姿态解算得出。
3.  **启动 ROS 2 验证节点**：
    加载手眼标定生成的 YAML 文件。
    ```bash
    ros2 run rj2506_vision verify_reprojection_error
    ```
4.  **执行空间运动捕获**：
    *   操作机械臂在验证板上方走出至少 **5 个**截然不同的极限姿态（左歪、右倒、大俯仰角，**务必引入大角度的 Pitch 和 Roll**），在每个姿态下停稳，按下回车键 `Enter` 捕获数据。

### 7.3 验证数学逻辑

在捕获的每一个姿态 $i$ 下，脚本将执行以下计算：

1.  **读取物理数据**：
    通过 TF 树读取那一瞬间法兰的物理位姿 $T_{base2flange\_i}$。
2.  **构建数学链路**：
    推算当前相机光心在世界坐标系（机器人基座）下的绝对位姿：
    $T_{cam\_world\_i} = T_{base2flange\_i} \cdot T_{flange2cam}$
3.  **三维到二维降维打击 (Re-projection)**：
    利用已知的相机内参矩阵 $K$ (3x3)，将 Excel 表格里的那 4 个物理 Ground Truth 坐标 $P_{world}^j$ 强行投射成理论的虚拟像素点 $s_{virtual}^{i, j}$：

    首先将世界坐标转换到相机坐标系下：
    $$ P_{cam}^{i, j} = (T_{cam\_world\_i})^{-1} \cdot P_{world}^j $$
    然后进行针孔透视投影（$X_c, Y_c, Z_c$ 为 $P_{cam}$ 的三个分量）：
    $$ u_{virtual} = f_x rac{X_c}{Z_c} + c_x, \quad v_{virtual} = f_y rac{Y_c}{Z_c} + c_y $$

4.  **计算均方根误差 (RMSE)**：
    在真实拍摄到的照片上，提取这 4 个圆点真实的像素坐标 $s_{real}^{i, j}$。
    **公式**：$RMSE = \sqrt{ rac{1}{M 	imes N} \sum_{i=1}^{M} \sum_{j=1}^{N} || s_{virtual}^{i, j} - s_{real}^{i, j} ||^2 }$
    *(其中 $M=5$ 为姿态数，$N=4$ 为点数)*

    **绝对合格线 (Pass)**：
    *   在这 $5 	imes 4 = 20$ 次比对中，平均重投影误差 $RMSE$ 必须 **$< 1.5$ 像素 (Pixels)**。
    *   如果**只有平移姿态时误差很小，一倾斜误差就急剧放大**，这是极其典型的“外参矩阵中平移向量是对的，但旋转矩阵完全算废了”的症状。此时必须让实施组退回重做任务二。

---
**结论**：只有这两套验证逻辑输出全局全绿的 `[PASS]` 标志，Phase 0 基建才能被称为“彻底夯实”。在这两项指标不达标之前，绝对不允许进入 Phase 1 的算法开发或 Phase 2 的深度学习训练，否则全是“ Garbage In, Garbage Out ”。
