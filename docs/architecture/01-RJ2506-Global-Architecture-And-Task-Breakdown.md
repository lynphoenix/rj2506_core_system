# RJ2506 全局架构与任务拆解总表

**文档状态**: V2.0 (已根据无硬件同步线现状调整)
**目标**: 提供整个双臂轮式机器人的全局需求、核心难点以及模块化的任务清单，作为整个项目文档树的**入口导航页**。

---

## 1. 核心需求与场景
*   **硬件载体**: RJ2506 差速底盘 + 双机械臂 + 平行夹爪 + 头部全局相机(OAK-D) + 手腕局部相机。
*   **计算平台**: NVIDIA Jetson Orin (由于现有任务冲突，Phase 0/1 阶段暂不刷入 PREEMPT_RT 实时内核)。
*   **业务场景**: 工厂半结构化环境下的纸箱堆垛、轮胎抱持、料片抓取。
*   **底层通信**: CANopen (1000Hz, 控制电机) + RS-485 (50Hz, 控制夹爪)。

## 2. 全局软件架构 (混合架构)
由于端到端模型 (ACT/DP) 无法保证导航和物理接触的绝对安全，且底盘停靠必然存在 $\pm 5cm$ 的宽容误差，系统采用**四段式控制管线**：

1.  **宏观导航 (Nav2)**: 负责厂区大范围移动，停靠在工站前 $\pm 5cm$ 的宽容区域内。
2.  **物理顺应补偿 (WBC)**: 放弃让底盘精调，利用双臂推拉料框等接触动作，配合阻抗控制，强行“抹平”底盘误差。
3.  **大尺度抓取 (ACT / Diffusion Policy)**: 输入图像与本体感知，输出 50 步双臂协作轨迹，解决反光和多片抓取难题。
4.  **微插闭环 (Virtual IBVS)**: 在距离目标 $15cm$ 时接管控制权。利用高空记忆的特征点构建“虚拟图像雅可比”，盲插闭环直到触底。

## 3. 全局任务拆解与文档导航

### Phase 0: 基础设施与物理真值采集 (Ground Truth)
这是不需要懂算法的实施人员必须做完的前置体力活。

👉 **[02-RJ2506-Overall-Workflow-SOP.md](./02-RJ2506-Overall-Workflow-SOP.md)** (面向实习生的傻瓜式物理执行手册，包含以下所有操作的终端命令与接线图，以及算法组验收标准)

*   **[0.1] 环境与连线**: 系统刷机（不含实时补丁）、ROS 2 安装、外设连通性测试。（注：因 OAK-D 标准版无触发引脚，放弃硬件连线同步）。
*   **[0.2] 相机外参标定**: 跑脚本算出 `camera_to_flange` 外参矩阵 ($T_{extrinsic}$)。
*   **[0.3] 绝对精度验证**: 制作物理探针(TCP)，手工戳出 4 个验证点的世界绝对坐标，与视觉计算对比，确保 RMSE < 1.5像素。
*   **[0.4] 盲插定妆照采集**: 人工将零件完美插入卡槽后抬高 15cm，拍摄作为后续 IBVS 控制的 2D 目标金标准 ($s_{target}^*$)。

### Phase 1: 核心硬骨头算法模块攻坚 (研发重点)
此阶段负责攻克卡脖子的技术难题，各模块可独立开发。

*   **[模块 5] C++ 守护进程与软件时间同步**: (优先级最高)
    *   👉 **[开发设计文档：03-Module5-Cpp-Daemon-Time-Sync.md](./03-Module5-Cpp-Daemon-Time-Sync.md)** (已完成软硬件妥协下的样条插值设计)
    *   *核心挑战*: 放弃硬件同步后，如何在 1000Hz 纯净线程里管理 CANopen，并利用**环形缓冲区 (Ring Buffer) + 样条插值**解决图像和电机的“因果混淆”问题。
*   **[模块 4] 虚拟 IBVS 视觉伺服**:
    *   👉 **[开发设计文档：04-Module4-Virtual-IBVS-Math.md](./04-Module4-Virtual-IBVS-Math.md)** (已完成数学推演与代码片段)
    *   *核心挑战*: 包含深度的 6x6 图像雅可比矩阵推导，防 Pitch/Roll 倾斜发散，伴随矩阵速度映射。
*   **[模块 3] 双目特征记忆提取器**:
    *   👉 **[开发设计文档：05-Module3-Feature-Memory.md](./05-Module3-Feature-Memory.md)** (待编写)
    *   *核心挑战*: 在恶劣光照下，从 15cm 高空的“定妆照”中绝对稳定地提取 $N \ge 4$ 个 3D 特征点 ($P_{world}^*$)。
*   **[模块 6] 力控与物理顺应 (Compliance)**:
    *   👉 **[开发设计文档：06-Module6-Impedance-Control.md](./06-Module6-Impedance-Control.md)** (待编写)
    *   *核心挑战*: 怎么在只支持位置/速度模式的 CANopen 驱动器外面，套一层阻抗控制模型（模拟弹簧），以及依靠 $\Delta I / \Delta t$ 识别插槽触底。

### Phase 2: AI 训练与全链路联调
*(待 Phase 1 的底层基建 100% 跑通并验证后开启)*
